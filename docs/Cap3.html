6.2 INTRODUÇÃO

Em qualquer análise de dados, antes de avançar para modelos estatísticos complexos, é fundamental conhecer os dados, entender sua distribuição, padrões e possíveis inconsistências.

Uma das formas mais eficientes de fazer isso é por meio de tabelas de resumo, que organizam os dados em uma estrutura fácil de interpretar, mostrando estatísticas descritivas como médias, medianas, contagens, proporções, desvios padrão, intervalos de confiança e testes de hipóteses básicos.

Tradicionalmente, criar tabelas de resumo pode ser um processo trabalhoso, especialmente quando lidamos com grandes bases de dados ou múltiplas variáveis. É preciso contar categorias, calcular proporções, organizar resultados e garantir que a formatação esteja adequada para relatórios ou publicações. Esse trabalho manual é não apenas tedioso, mas também propenso a erros.

É nesse contexto que surgem as tabelas automáticas, ferramentas que permitem gerar resumos estatísticos completos e esteticamente padronizados, economizando tempo e aumentando a confiabilidade das análises. Com tabelas automáticas, é possível:

  • Resumir variáveis qualitativas (categóricas) com contagens e proporções
  • Resumir variáveis quantitativas com médias, medianas, desvios-padrão, quartis e intervalos.
  • Integrar análises de associação entre variáveis de forma automatizada com testes de hipótese.
  • Gerar relatórios padronizados para artigos científicos, apresentações ou dashboards.
  • Facilitar a reprodutibilidade: se os dados mudarem, as tabelas podem ser atualizadas automaticamente sem refazer cálculos manualmente.

6.3 O PACOTE gtsummary

O pacote gtsummary no R surge exatamente para suprir essa necessidade. Ele permite criar tabelas automáticas com análises descritivas e inferências básicas, tanto para variáveis qualitativas quanto quantitativas, com poucas linhas de código, e já fornece recursos para:

  • Personalizar rótulos e títulos.
  • Adicionar testes estatísticos automáticos (como qui-quadrado ou t-test).
  • Adicionar intervalos de confiança.
  • Produzir tabelas prontas para publicação, em Word, HTML ou PDF.

O gtsummary é amplamente utilizado em pesquisas clínicas, epidemiológicas, sociais e empresariais, mas seu uso pode ser estendido a qualquer contexto em que seja necessário resumir dados com precisão, eficiência e estética.

Neste treinamento, vamos explorar como criar tabelas automáticas completas no R, cobrindo desde o resumo de variáveis simples até a geração de tabelas cruzadas e testes de associação. O objetivo é que você consiga:

  1. Criar tabelas descritivas para variáveis qualitativas e quantitativas
  2. Automatizar a análise de associação entre variáveis
  3. Produzir resultados claros e padronizados, prontos para relatórios e publicações

Para facilitar o aprendizado e garantir uma progressão natural no uso do gtsummary, este treinamento foi dividido em duas etapas. A ideia é começar pelo essencial, explorando primeiro as variáveis qualitativas, e avançar gradualmente até análises mais completas que envolvem variáveis quantitativas também.

  1. Variáveis qualitativas:
    Nesta primeira etapa, vamos aprender a criar tabelas de frequências e proporções, compreender como o gtsummary organiza e apresenta informações categóricas e, em seguida, incluir testes de associação (como o teste qui-quadrado e exato de Fisher). Esse primeiro contato permitirá que você se familiarize com a lógica e as principais funções do pacote.

  2. Variáveis qualitativas e quantitativas: Em seguida, ampliaremos a complexidade das tabelas, combinando variáveis de diferentes naturezas. Vamos explorar resumos numéricos (como média, mediana e desvio-padrão), além de aprender a apresentar resultados integrados — por exemplo, comparar uma variável quantitativa entre categorias qualitativas — de forma automática, elegante e pronta para publicação. Essa comparação poderá ser feita a partir de testes apropriados (Teste-t, ANOVA e afins)

Antes de construir sua tabela, é importante se atentar sobre a linguagem em que a tabela será constuída. Para alterar a linguagem da sua tabela, execute um dos seguintes comandos no começo do seu script:

Código 
theme_gtsummary_language("en", big.mark = ",", decimal.mark = ".") # formatação em inglês (ponto pra decimais e vírgula para milhares)
theme_gtsummary_language("pt", big.mark = ".", decimal.mark = ",") # formatação em português (vírgula pra decimais e ponto para milhares)

A função responsável por criar tabelas a partir do gtsummary é a função tbl_summary. Essa função tem muitos argumentos e, ao longo do treinamento, veremos os argumentos mais utilizados.

6.4 VARIÁVEIS QUALITATIVAS

Começaremos nossa prática com as variáveis qualitativas, também chamadas de categóricas. Elas representam grupos, categorias ou classificações, e não possuem uma escala numérica natural — por exemplo: sexo, estado civil, presença ou ausência de uma doença, nível de escolaridade, região de residência etc.

Resumir variáveis qualitativas é uma das tarefas mais frequentes em análises de dados, especialmente em estudos clínicos, sociais e empresariais. Normalmente, desejamos saber quantos indivíduos pertencem a cada categoria e qual é a proporção de cada grupo em relação ao total da amostra e avançar para investigação entre a associação entre as variáveis.

6.4.3 INVESTIGANDO ASSOCIAÇÃO

Este tópico será destinado ao uso do teste qui-quadrado de Pearson, o teste exato de Fisher e o coeficiente de Cramer na investigação da associação entre duas variáveis categóricas

6.4.3.1 TESTE QUI-QUADRADO DE INDEPENDÊNCIA

O teste qui-quadrado de independência é feito com a função chisq.test a partir do seguinte comando

Teste = chisq.test(x=VariavelLinha,y=VariavelColuna)

Este objeto armazena o valor da estatística de teste X-squares, os graus de liberdade df e o valor-p p-value.

Com esse objeto, podemos extrair os valores esperados sob independência (util para verificar a suposição do teste):

Alem disso, é possível extrair os resíduos padronizados:

Teste$stdres

Estes resíduos são utilizados para detectar as combinações entre as categorias que foram influentes para a significância do teste (quando o teste é significativo)

6.4.3.2 TESTE EXATO DE FISHER

O teste exato de Fisher é feito a partir do seguinte comando

Tabela = table(VariavelLinha,VariavelColuna)
Teste = fisher.test(Tabela)

O objeto Teste armazena o valor-p na saída p-value.

6.4.3.3 COEFICIENTE DE CRAMER

O coeficiente de Cramer pode ser calculado com o pacote rstatix a partir dos seguintes comandos:

Tabela = table(VariavelLinha,VariavelColuna)
cramer_v(Tabela)

O resultado é exatamente o valor do coeficiente entre 0 e 1.

6.5 VARIÁVEIS QUANTITATIVAS

Agora avançaremos para as variáveis quantitativas. Diferentemente das qualitativas, essas variáveis representam valores numéricos mensuráveis, geralmente associados a alguma escala — por exemplo: idade, peso, altura, renda, pressão arterial, tempo de espera, número de filhos, entre outros.

Essas variáveis permitem não apenas identificar valores individuais, mas também resumir o comportamento geral dos dados por meio de medidas estatísticas (medidas-resumo).

Resumir variáveis quantitativas é fundamental em praticamente qualquer análise de dados. Em vez de contar categorias, nosso objetivo aqui é entender:

  • Onde os dados se concentram (medidas de tendência central, como média e mediana);
  • Quão dispersos eles são (desvio padrão, intervalo interquartil, mínimo e máximo);
  • E, muitas vezes, comparar esses resumos entre grupos, o que nos leva naturalmente a investigações inferenciais (testes de hipóteses).

## **INTRODUÇÃO**

Em qualquer **análise de dados**, antes de **avançar para modelos estatísticos complexos**, é fundamental **conhecer os dados**, entender sua **distribuição**, **padrões** e **possíveis inconsistências**.  

Uma das formas mais **eficientes** de fazer isso é por meio de **tabelas de resumo**, que **organizam os dados** em uma estrutura **fácil de interpretar**, mostrando **estatísticas descritivas como médias, medianas, contagens, proporções, desvios padrão, intervalos de confiança e testes de hipóteses básicos**.

Tradicionalmente, criar **tabelas de resumo pode ser um processo trabalhoso**, especialmente quando lidamos com **grandes bases de dados ou múltiplas variáveis**. É preciso **contar categorias, calcular proporções, organizar resultados e garantir que a formatação esteja adequada para relatórios ou publicações**. Esse trabalho **manual** é não apenas **tedioso**, mas também propenso a **erros**.

É nesse contexto que surgem as **tabelas automáticas**, ferramentas que permitem gerar resumos estatísticos completos e esteticamente padronizados, **economizando tempo e aumentando a confiabilidade das análises**. Com tabelas automáticas, é possível:

- Resumir **variáveis qualitativas** (categóricas) com contagens e proporções  
- Resumir **variáveis quantitativas** com médias, medianas, desvios-padrão, quartis e intervalos.
- Integrar **análises de associação** entre variáveis de forma automatizada com testes de hipótese.
- Gerar **relatórios padronizados** para artigos científicos, apresentações ou dashboards.  
- Facilitar a **reprodutibilidade**: se os dados mudarem, as tabelas podem ser atualizadas automaticamente sem refazer cálculos manualmente.

## **VARIÁVEIS QUALITATIVAS**

Começaremos nossa prática com as **variáveis qualitativas**, também chamadas de **categóricas**. #### **TESTE QUI-QUADRADO DE INDEPENDÊNCIA**

O teste qui-quadrado de independência é feito com a função `chisq.test` a partir do seguinte comando

```r
Teste = chisq.test(x=VariavelLinha,y=VariavelColuna)
```

Este objeto armazena o valor da estatística de teste `X-squares`, os graus de liberdade `df` e o valor-p `p-value`.

Com esse objeto, podemos extrair os valores esperados sob independência (util para verificar a suposição do teste):

Alem disso, é possível extrair os resíduos padronizados:

```r
Teste$stdres
```

Estes resíduos são utilizados para detectar as combinações entre as categorias que foram influentes para a significância do teste (quando o teste é significativo)

#### **TESTE EXATO DE FISHER**

O teste exato de Fisher é feito a partir do seguinte comando

```r
Tabela = table(VariavelLinha,VariavelColuna)
Teste = fisher.test(Tabela)
```

O objeto Teste armazena o valor-p na saída `p-value`.

#### **COEFICIENTE DE CRAMER**

O coeficiente de Cramer pode ser calculado com o pacote `rstatix` a partir dos seguintes comandos:

```r
Tabela = table(VariavelLinha,VariavelColuna)
cramer_v(Tabela)
```

O resultado é exatamente o valor do coeficiente entre 0 e 1.

> O teste de Tukey é realizado a partir do comando `TukeyHSD` e é a comparação múltipla mais usada após uma ANOVA.

> O teste de Games Howell é realizado a partir do comando `games_howell_test` e é a comparação múltipla mais usada após uma ANOVA de Welch.

Para **as variáveis em que a hipótese nula** da ANOVA (usual ou de Welch) não foi rejeitada, todas **as letras são iguais**. 

A ideia é **incluir uma coluna à direita na tabela final trazendo** essas letras que são resultados das **comparações múltiplas**. Agora, podemos **construir a nossa tabela final**:

```{r}
comparacoes_letras <- function(data, variable, by, ...) {
  letras <- data.frame(
    variavel = c("Idade", "Distancia",
                 "Atraso_Partida", "Atraso_Chegada"),
    comparacao = c(
      "Insatisfeito = a<br>Neutro = a<br>Satisfeito = b",
      "Insatisfeito = a<br>Neutro = b<br>Satisfeito = c",
      "Insatisfeito = a<br>Neutro = a<br>Satisfeito = a",
      "Insatisfeito = a<br>Neutro = a<br>Satisfeito = a"
    ),
    stringsAsFactors = FALSE  
  )
  result <- letras$comparacao[letras$variavel == variable]
  tibble::tibble(`**Comparações Múltiplas**` = result)
}

tbl_summary(data = Dados2,
            by = Servico_Online,
            statistic = all_continuous() ~ "{mean} ({sd})",
            label = list(
              Idade ~ "Idade<sup>A</sup>",
              Distancia ~ "Distância<sup>W</sup>",
              Atraso_Partida ~ "Atraso na partida<sup>A</sup>",
              Atraso_Chegada ~"Atraso na chegada<sup>A</sup>")
)%>%
  add_p(
    test = everything() ~ "oneway.test",
    test.args = list(
      c(Idade, Atraso_Partida, Atraso_Chegada) ~ list(var.equal = T),   # Variâncias iguais (t de Student)
      Distancia ~ list(var.equal = F)  # Variâncias diferentes (t de Welch)
    ),

    fns = everything() ~ comparacoes_letras,
    location = everything() ~ "label"
  ) %>%
  as_gt() %>%                   
  fmt_markdown(columns = c(label,`**Comparações Múltiplas**`))%>%

    column_labels.font.size = "22px"
  )
```
